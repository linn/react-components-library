# DatePicker docs

This component is really just a wrapper of the material-ui [DatePicker](https://mui.com/x/react-date-pickers/) component (using moment.js as our date library), with some slight adaptations for style and convenience.

As such it must always be rendered within a LocalizationProvider, i.e.

```<LocalizationProvider dateAdapter={AdapterMoment}>```

It takes the following props:

| Prop        |Type                       | Required/Default | Narrative
| -----------|----------------------------|------------------|------------------------------------------------------------------------            
| value      | iso string or moment.js object | defaults to null  | the current value of the Picker
| minDate    | iso string or moment.js object | defaults to null  | the minimum possible value of the Picker
| maxDate    | iso string or moment.js object | defaults to null  | the maximum possible value of the Picker
| required   | bool                           | defaults to false | whether a red asterisk is rendered to indicate the field is required
| label      | string                         | defaults to ''    | the label string to show above the picker
| disabled   | bool                           | defaults to false | whether the field is disabled for input
| onChange   | func                           | defaults to empty function | the function to be called on the newValue when the value of the picker is changed. Note that the passed newValue parameter will always be of the 'moment' object type, not an iso string, regardless of the type of the current value prop.
