# usePreviousNextNavigation

A hook that provides previous/next navigation through a list of search results. It reads the current item's ID from the route (either a URL param or a query-string parameter) and returns callbacks to navigate to the adjacent items.

## Signature

```js
const [goPrev, goNext, prevResult, nextResult, currentResult] = usePreviousNextNavigation(
    urlBuilder,
    searchResults,
    navigateFunction,
    idStyle,
    idFieldName
);
```

## Basic usage

```jsx
import { useNavigate } from 'react-router-dom';
import usePreviousNextNavigation from '@linn-it/linn-form-components-library/hooks/usePreviousNextNavigation';

function ProductDetail({ searchResults }) {
    const navigate = useNavigate();

    const [goPrev, goNext] = usePreviousNextNavigation(
        item => `/products/${item}`,
        searchResults,
        navigate
    );

    return (
        <>
            <button onClick={goPrev} disabled={!goPrev}>Previous</button>
            <button onClick={goNext} disabled={!goNext}>Next</button>
        </>
    );
}
```

## With query-string IDs

When the current item ID lives in a query-string parameter rather than a route param, pass `'query'` and the field name:

```jsx
const [goPrev, goNext] = usePreviousNextNavigation(
    item => `/orders?orderId=${item}`,
    searchResults,
    navigate,
    'query',    // idStyle
    'orderId'   // idFieldName — reads ?orderId= from the current URL
);
```

## Showing adjacent item details

`prevResult` and `nextResult` let you preview what the user will navigate to:

```jsx
const [goPrev, goNext, prevResult, nextResult, currentResult] = usePreviousNextNavigation(
    item => `/items/${item}`,
    searchResults,
    navigate
);

return (
    <>
        {prevResult && <small>← {prevResult}</small>}
        <h1>Current: {currentResult}</h1>
        {nextResult && <small>{nextResult} →</small>}
        <button onClick={goPrev} disabled={!goPrev}>Prev</button>
        <button onClick={goNext} disabled={!goNext}>Next</button>
    </>
);
```
